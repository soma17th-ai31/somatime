## 3. 환경 변수

`backend/.env.example`:
```bash
# === Backend ===
DATABASE_URL=sqlite:///./somameet.db
SESSION_SECRET=change-this-32-char-random-string
APP_BASE_URL=http://localhost:5173

# === LLM (Upstage solar-pro3, OpenAI 호환 SDK) ===
LLM_PROVIDER=upstage              # upstage | template
UPSTAGE_API_KEY=
UPSTAGE_BASE_URL=https://api.upstage.ai/v1
UPSTAGE_MODEL=solar-pro3
UPSTAGE_TIMEOUT_SECONDS=45
# template: API 호출 없이 결정적 문자열 템플릿. 키 발급 전 우회용.
```

`frontend/.env.local.example`:
```bash
VITE_API_BASE_URL=http://localhost:8000
```

**키 없을 때 거동:**
- `LLM_PROVIDER=template` 로 두면 `TemplateAdapter` 호출되어 흐름은 통과 (S11 privacy 자동 통과)
- 실제 Upstage live 호출 privacy 검증은 키 받은 뒤 다시 한 번 실행 필요

**제거된 환경변수 (v1 → v3):**
- `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `GOOGLE_REDIRECT_URI`, `GOOGLE_OAUTH_SCOPES` (Q3)
- `GEMINI_API_KEY`, `ANTHROPIC_API_KEY`, `OPENAI_API_KEY` — Upstage 단일 provider

## 4. DB 스키마

SQLite + SQLAlchemy 2.x + Alembic.

### 4.1 `meetings`
| 컬럼 | 타입 | 제약 | 설명 |
|---|---|---|---|
| id | INTEGER | PK AUTOINCREMENT | |
| slug | VARCHAR(8) | UNIQUE NOT NULL | base62, 공개 |
| organizer_token | VARCHAR(32) | NOT NULL | URL-safe random, 비밀 |
| title | VARCHAR(200) | NOT NULL | |
| date_mode | VARCHAR(8) | NOT NULL | **신규** Q5. `range` 또는 `picked` |
| date_range_start | DATE | NULLABLE | range 모드일 때만 사용 |
| date_range_end | DATE | NULLABLE | range 모드일 때만 사용 |
| candidate_dates | JSON | NULLABLE | **신규** Q5. picked 모드일 때 ISO date 문자열 배열. 예: `["2026-05-07","2026-05-09","2026-05-10"]` |
| duration_minutes | INTEGER | NOT NULL | 30분 배수 |
| participant_count | INTEGER | NOT NULL | **목표 인원**. Q2 calculate 게이트 기준 |
| location_type | VARCHAR(10) | NOT NULL | **Q6** `online` / `offline` / `any` |
| offline_buffer_minutes | INTEGER | NOT NULL DEFAULT 30 | **신규** Q8. 30 / 60 / 90 / 120 |
| time_window_start | TIME | NOT NULL | 기본 09:00 |
| time_window_end | TIME | NOT NULL | 기본 22:00 |
| include_weekends | BOOLEAN | NOT NULL DEFAULT FALSE | |
| confirmed_slot_start | DATETIME | NULLABLE | 확정 후 채워짐 |
| confirmed_slot_end | DATETIME | NULLABLE | |
| confirmed_share_message | TEXT | NULLABLE | **신규** Q9. confirm 시 프론트가 보낸 share_message_draft 저장 |
| created_at | DATETIME | NOT NULL | |

**날짜 일관성 제약 (애플리케이션 레벨):**
- `date_mode = 'range'` → `date_range_start`/`end` NOT NULL, `candidate_dates` NULL
- `date_mode = 'picked'` → `date_range_start`/`end` NULL, `candidate_dates` 길이 ≥ 1

### 4.2 `participants`
| 컬럼 | 타입 | 제약 | 설명 |
|---|---|---|---|
| id | INTEGER | PK | |
| meeting_id | INTEGER | FK meetings(id), CASCADE | |
| nickname | VARCHAR(50) | NOT NULL | |
| token | VARCHAR(32) | NOT NULL UNIQUE | participant 쿠키 발급용 |
| pin | VARCHAR(8) | NULLABLE | **신규** Q7. **비암호화 평문 PIN(4자리 숫자 권장).** 입력 시에만 저장. NULL이면 PIN 없음 |
| source_type | VARCHAR(10) | NULLABLE | ics / manual (`google` 제거) |
| confirmed_at | DATETIME | NULLABLE | 입력 완료 시점. **Q2 calculate 게이트에서 NOT NULL 카운트** |
| created_at | DATETIME | NOT NULL | |

UNIQUE INDEX `(meeting_id, nickname)`.

> **Q7 보안 메모**: PIN을 평문으로 저장한다. MVP 단순화 결정. 운영 단계로 가면 bcrypt 해시로 전환해야 함. 절대 운영 DB 덤프를 외부에 노출하지 말 것. README의 보안 섹션에 이 한계를 명시할 것.

## 5. API 엔드포인트

전부 `/api` 프리픽스. 응답은 모두 JSON. 에러는 `{ "error_code": "...", "message": "...", "suggestion": "..." }` 형식.

| # | Method | Path | 권한 | 설명 |
|---|---|---|---|---|
| 1 | POST | `/api/meetings` | 누구나 | 회의 생성 (date_mode/buffer/location 포함) |
| 2 | GET | `/api/meetings/{slug}` | URL만 | 회의 정보 + 응답 진행률 + QR용 share_url |
| 3 | POST | `/api/meetings/{slug}/participants` | 누구나 | 닉네임 + (optional) PIN 등록 + 쿠키 발급 |
| 3b | POST | `/api/meetings/{slug}/participants/login` | 누구나 | **신규** PIN 보유 시 닉네임+PIN 으로 재진입 (쿠키 재발급) |
| 4 | POST | `/api/meetings/{slug}/availability/manual` | 본인 쿠키 | 직접 입력 |
| 5 | POST | `/api/meetings/{slug}/availability/ics` | 본인 쿠키 | ICS 업로드 (multipart) |
| 6 | GET | `/api/meetings/{slug}/timetable` | URL만 | 시간대별 가용 인원 (가로 레이아웃 데이터) |
| 7 | POST | `/api/meetings/{slug}/calculate` | URL만 (응답 진행률 충족 필요) | **deterministic only**. windows + 단순 ranking 후보 3개. **LLM 호출 없음** |
| 8 | POST | `/api/meetings/{slug}/recommend` | URL만 (응답 진행률 충족 필요) | **신규 Q9**. LLM 1회로 후보별 reason + share_message_draft 동시 생성. 검증 실패 시 최대 3회 재호출 |
| 9 | POST | `/api/meetings/{slug}/confirm` | **organizer 토큰** | 확정. body의 `share_message_draft` 그대로 저장. **LLM 호출 없음** |

**v1 → v3 제거:**
- ~~`GET /api/meetings/{slug}/availability/google/oauth-url`~~ (Q3)
- ~~`GET /api/auth/google/callback`~~ (Q3)

```jsonc

```

**POST /api/meetings/{slug}/calculate** (deterministic only, LLM 호출 없음)
```jsonc
// 응답자 < target → 409
{
  "error_code": "insufficient_responses",
  "message": "아직 모든 참여자가 일정을 제출하지 않았습니다.",
  "current": 2, "required": 4
}

// 충족 시 → 200
{
  "summary": null,                         // calculate에선 LLM 호출 X 이므로 항상 null
  "best_available_count": 4,
  "total_participants": 4,
  "candidates": [                          // deterministic ranking
    {
      "start": "2026-05-12T14:00:00+09:00",

## 6. 접근 제어

### URL 구조
```
주최자 전용:  /m/{slug}?org={organizer_token}
참여자 공유용: /m/{slug}
```

### 토큰 발급
- `slug`: 8자 base62 (예: `aB3kF9xQ`)
- `organizer_token`: `secrets.token_urlsafe(24)` → 32자 이상
- `participant_token`: 참여자 등록 시 발급, **HttpOnly 쿠키** `somameet_pt_{slug}` 로 저장
- `pin` (optional): 평문 4자리 숫자. 등록 시 입력하면 저장, 재진입 시 닉네임+PIN으로 쿠키 재발급

### 권한 매트릭스

| Endpoint | 필요 권한 | 검증 방법 |
|---|---|---|
| `POST /meetings` | 누구나 | - |
| `GET /meetings/{slug}` | URL만 | - |
| `POST /participants` | 누구나 | 응답 시 쿠키 set, PIN 입력하면 저장 |
| `POST /participants/login` | 누구나 (PIN 보유자) | 닉네임+PIN 매칭 → 쿠키 set |
| `POST /availability/*` | 본인 participant | 쿠키 |
| `GET /timetable` | URL만 | - |
| `POST /calculate`, `POST /recommend` | URL만 (응답 진행률 충족) | submitted_count == target_count 필요, 미충족 시 409 |
| `POST /confirm` | **주최자만** | `X-Organizer-Token` 헤더 |

### 프론트 UX 규칙
- 회의 생성 직후 결과 화면:
  ```
  [내 관리용 링크 — 공유 금지]  /m/aB3kF9xQ?org=...   [복사]
  [팀원에게 공유할 링크]         /m/aB3kF9xQ           [복사]  [QR 코드 펼치기]
  ```
- QR은 `share_url` 만 표기 (organizer_token 포함된 url 절대 QR로 만들지 말 것)
- 주최자 URL로 들어오면 "확정" 버튼 노출
- 참여자 URL로 들어오면 "확정" 버튼 미노출 (하드 가드는 백엔드)
- **Q2**: `submitted_count < target_count` 동안 "결과 보기"/"추천받기" 버튼 disabled + 진행률 표시
- **Q9**: "결과 보기" 클릭 시 calculate 호출(deterministic candidates만 표시), "추천받기" 별도 버튼이 recommend 호출 (LLM 호출은 이때만)
- **Q7**: 참여자 페이지에 "이미 닉네임 등록함, PIN으로 재진입" 보조 폼 노출

## 8. LLM 사용 경계 (Q9)

| 작업 | LLM(Upstage) 사용 |
|---|---|
| 공통 가능 시간 enumeration | ❌ deterministic |
| candidate_windows 생성 (회의 길이/버퍼/시간대 적용) | ❌ deterministic |
| `POST /calculate` 후보 ranking | ❌ deterministic |
| **`POST /recommend` — 후보 선택** | ✅ Upstage |
| **`POST /recommend` — 후보별 reason** | ✅ Upstage |
| **`POST /recommend` — 후보별 share_message_draft** | ✅ Upstage (같은 호출에서 함께 생성) |
| **`POST /recommend` — summary** | ✅ Upstage |
| 응답 후 검증 | ❌ deterministic |
| 검증 실패 시 fallback ranking | ❌ deterministic |
| `POST /confirm` 메시지 | ❌ — 프론트가 보낸 값 그대로 저장 |
| ICS 파싱 | ❌ deterministic |
| 입력 검증 | ❌ Pydantic + 단순 체크 |

**LLM 호출 횟수 제한 (Q9):**
- 회의 1건당 최대 **4회** = recommend 1차 + 재시도 최대 3회
- 4회 안에 검증 통과 못 하면 `deterministic_fallback_candidates` 사용 (응답 `source: "deterministic_fallback"`)
- 재시도는 검증 실패 시에만 (네트워크 오류는 별도 — 즉시 fallback)

**Privacy 가드:**
- payload 에는 닉네임 + busy/free 시간 블록 + 회의 메타데이터만
- 이벤트 제목/설명/위치 절대 미포함
- privacy 검증은 인수 시나리오 S11 에서 spy 또는 live 로 확인

from abc import ABC, abstractmethod
from dataclasses import dataclass
from datetime import datetime
from typing import List

from app.db.models import Meeting
from app.services.scheduler import CandidateWindow


@dataclass(frozen=True)
class Slot:
    start: datetime
    end: datetime


class LLMAdapter(ABC):
    @abstractmethod
    def recommend(
        self,
        candidate_windows: List[CandidateWindow],
        meeting: Meeting,
        max_candidates: int = 3,
    ) -> dict:
        """1회 호출로 summary + candidates(reason + share_message_draft 포함) 모두 생성.

        Returns:
            {
              "summary": str,
              "candidates": [
                {
                  "start": str (ISO+09:00),
                  "end": str (ISO+09:00),
                  "reason": str,
                  "share_message_draft": str
                }, ...
              ]
            }
        """

    def build_recommendation_payload(
        self,
        candidate_windows: List[CandidateWindow],
        meeting: Meeting,
        max_candidates: int,
    ) -> dict:
        """Privacy-safe payload. ONLY: title, location_type, duration_minutes, candidate_windows."""
        return {
            "meeting": {
                "title": meeting.title,
                "location_type": meeting.location_type,
                "duration_minutes": meeting.duration_minutes,
                "target_participants": meeting.participant_count,
                "offline_buffer_minutes": meeting.offline_buffer_minutes,
            },
            "rules": {
                "slot_unit_minutes": 30,
                "max_candidates": max_candidates,
            },
            "candidate_windows": [
                {
                    "start": w.start.isoformat(),
                    "end": w.end.isoformat(),
                    "available_count": w.available_count,
                    "is_full_match": w.is_full_match,
                    "available_participants": list(w.available_nicknames),
                    "unavailable_participants": list(w.missing_participants),
                }
                for w in candidate_windows
            ],
        }
```

```python
import json
import os
from typing import List

from openai import OpenAI

from app.services.llm.base import LLMAdapter
from app.services.llm.prompts import SYSTEM_PROMPT_RECOMMEND, build_recommendation_user_prompt


class UpstageAdapter(LLMAdapter):
    def __init__(self):
        api_key = os.environ.get("UPSTAGE_API_KEY")
        if not api_key:
            raise RuntimeError("UPSTAGE_API_KEY not set; use LLM_PROVIDER=template instead")
        self.client = OpenAI(
            api_key=api_key,
            base_url=os.environ.get("UPSTAGE_BASE_URL", "https://api.upstage.ai/v1"),
            timeout=float(os.environ.get("UPSTAGE_TIMEOUT_SECONDS", "45")),
        )
        self.model = os.environ.get("UPSTAGE_MODEL", "solar-pro3")

    def recommend(self, candidate_windows, meeting, max_candidates=3):
        payload = self.build_recommendation_payload(candidate_windows, meeting, max_candidates)
        response = self.client.chat.completions.create(
            model=self.model,
            temperature=0.2,
            messages=[
                {"role": "system", "content": SYSTEM_PROMPT_RECOMMEND},
                {"role": "user", "content": build_recommendation_user_prompt(payload)},
            ],
        )
        content = response.choices[0].message.content or "{}"
        return json.loads(content)
```

### 8.2 recommend 엔드포인트 흐름

```python
# backend/app/api/recommend.py 의사코드

def recommend(slug):
    meeting = get_meeting(slug)

    submitted = count_submitted_participants(meeting.id)
    if submitted < meeting.participant_count:
        return 409, {"error_code": "insufficient_responses",
                     "current": submitted, "required": meeting.participant_count}

    busy = load_busy_blocks(meeting.id)
    windows = generate_candidate_windows(meeting, busy)

    if not windows:
        return 200, {"candidates": [], "summary": None,
                     "suggestion": "회의 길이를 줄이거나 날짜 범위를 넓혀보세요.",
                     "source": "deterministic_fallback", "llm_call_count": 0}

    adapter = get_llm_adapter()
    last_error = None
    for attempt in range(1, 5):                  # Q9 — 최대 4회
        try:
            llm_output = adapter.recommend(windows, meeting)
            candidates = validate_and_enrich(llm_output["candidates"], windows, meeting)
            return 200, {**llm_output, "candidates": candidates,
                         "source": "llm", "llm_call_count": attempt}
        except (CandidateValidationError, json.JSONDecodeError) as exc:
            last_error = str(exc)
            continue
        except Exception:                          # 네트워크 등 — 즉시 fallback
            break

    fallback = deterministic_top_candidates(windows)
    return 200, {
        "summary": "LLM 추천 검증을 4회 시도해도 통과하지 못해 결정적 알고리즘으로 후보를 선정했습니다.",
        "candidates": [{**c.model_dump(),
                        "share_message_draft": render_template_share_message(meeting, c)}
                       for c in fallback],
        "source": "deterministic_fallback",
        "llm_call_count": 4 if last_error else 0,
    }
```

## 10. 인수 기준 시나리오

위임받은 쪽이 `backend/tests/acceptance/test_*.py` 에 그대로 구현. **전부 통과해야 MVP 완성.**

### S1. Happy path
- **Given** 주최자 A, 참여자 B/C/D, target_count=4, date_mode=range, location=offline, buffer=60
- **When** A가 회의 생성 → A/B/C/D 모두 입력 → A가 calculate → A가 recommend → 첫 후보 confirm (recommend의 share_message_draft 동봉)
- **Then**
  - 회의 생성 status 201, slug 길이 8, organizer_token 길이 32+
  - calculate 200, source="deterministic", reason=null
  - recommend 200, source="llm", llm_call_count==1, candidates 길이 ≤ 3, 모두 4명 교집합, share_message_draft 비어있지 않음
  - confirm 200, confirmed_share_message가 body로 받은 값과 동일
  - 어디에도 일정 제목/장소/설명 단어 없음

### S1b. calculate/recommend 게이트 (Q2)
- target_count=4, A/B만 제출
- calculate / recommend 둘 다 **409** `insufficient_responses` (current:2, required:4)
- GET /meetings/{slug} 응답에 `is_ready_to_calculate: false`

### S2. 가변 오프라인 버퍼 (Q8)
- offline 회의, 60분, 1명이 12:00-13:30 + 14:30-16:00 busy
- buffer=30 → 13:30-14:30 슬롯 **제외**
- buffer=60 → 13:30-14:30 슬롯 **제외** (양옆 60분 위반)
- buffer=120 → 더 넓은 영역도 **제외**
- location=any + buffer=30 → offline과 동일하게 **제외**
- location=online → buffer 무시, **포함**

### S3. 후보 0개 → "1명 빠진" 폴백
- 4명 중 1명이 후보 기간 전부 busy
- candidate_windows에 `is_full_match=false, available_count=3` 항목 포함
- LLM 또는 fallback이 그 중에서 선택
- 응답 후보에 `missing_participants: ["닉네임"]`

### S4. 폴백조차 0개 → 조건 완화

### S13. **신규** — recommend LLM 호출 횟수 (Q9)

### S14. **신규** — PIN optional 재진입 (Q7)

- DB의 pin 컬럼은 평문 "1234" 그대로 (해시 아님)

- 2026-05-08 (사이의 빠진 날) 슬롯이 후보에 **없음**
