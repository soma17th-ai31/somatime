# 박세종 기획서 초안

## 1. 문제 정의 및 프로젝트 개요

### 프로젝트 한 줄 정의

SW마에스트로 구성원들이 개인 일정 세부 내용을 공개하지 않고도 여러 사람의 공통 가능 시간을 빠르게 찾을 수 있도록 돕는 개인정보 보호 일정 조율 Agentic Workflow 프로젝트입니다.

### 서비스 한 줄 정의

여러 사람이 가능한 시간대만 제공하면, Agent가 공통 후보 시간을 계산하고 가장 적절한 회의 시간을 추천해주는 일정 조율 서비스입니다.

### 서비스 선정 배경

SW마에스트로에서는 팀 회의, 멘토링, 엑스퍼트 상담, 코칭처럼 여러 사람이 함께 시간을 맞춰야 하는 상황이 반복됩니다. 현재는 대부분 메신저로 "언제 가능하세요?"를 반복하며 조율하고, 참여자가 늘어날수록 응답 정리와 재확인이 번거로워집니다.

공유캘린더는 편리하지만 일정 제목, 장소, 세부 내용이 노출될 수 있습니다. 또한 각자가 Google Calendar, Apple Calendar, Outlook, 네이버 캘린더, 수기 일정표 등 서로 다른 도구를 쓰기 때문에 특정 캘린더 하나에 의존하기 어렵습니다.

### 해결하려는 문제

- 메신저 기반 일정 조율은 응답이 흩어져 정리 시간이 오래 걸립니다.
- 공유캘린더는 개인 일정의 제목, 장소, 내용이 드러날 수 있습니다.
- 사용자마다 사용하는 캘린더 도구가 달라 일정을 공통 형식으로 맞추기 어렵습니다.
- 일정 조율은 본질적인 개발/기획 작업은 아니지만 팀의 집중 시간을 계속 소모합니다.

### 대상 사용자

1차 대상 사용자는 SW마에스트로에서 3-6명 규모의 회의 시간을 반복적으로 조율해야 하는 연수생입니다.

### 핵심 가치

개인 일정의 세부 내용은 숨기고, 회의 조율에 필요한 busy/free 정보만 활용해 공통 가능 시간을 빠르게 찾는 것입니다.

## 2. 사용자 및 Agent 설계

### Agent의 역할

- 회의명, 소요 시간, 날짜 범위, 참여자, 필수 참석자 조건을 구조화합니다.
- 직접 입력, `.ics`, CSV/텍스트 등 서로 다른 일정 입력을 공통 availability schema로 변환합니다.
- 일정 제목, 장소, 설명은 제거하고 시작/종료 시간과 busy/free 정보만 사용합니다.
- 공통 가능 시간을 계산한 뒤 후보 3개를 추천합니다.
- 추천 이유와 복사용 확정/재조율 메시지를 생성합니다.

### Agent의 자율성 범위

Agent는 후보 시간 추천과 메시지 작성까지만 수행합니다. 최종 선택과 공유는 사람이 합니다.

## 3. 핵심 기능 및 사용자 흐름

### 주요 사용자 시나리오

팀장이 팀원 3명과 멘토 1명의 1시간 멘토링 시간을 잡아야 합니다. 각 참여자는 개인 일정 세부 내용 없이 가능한 시간 또는 바쁜 시간만 제공합니다. SomaTime은 공통 가능 시간을 계산하고, 적절한 후보 3개와 추천 이유를 보여줍니다.

### 핵심 기능

1. 회의 조건 입력
2. 개인 일정 가져오기 및 정규화
3. 공통 가능 시간 탐색
4. 후보 시간 추천
5. 공유 메시지 생성

## 4. 기술 구현 설계

### 기술 아이디어

- Frontend/Demo UI: Streamlit
- Backend API: FastAPI
- Agent Workflow: LangGraph
- Data validation: Pydantic
- Calendar parsing: `icalendar` 또는 CSV parser
- Storage: SQLite 또는 로컬 JSON 파일
- Scheduling engine: Python deterministic time-slot calculation module

### 핵심 구조

```text
사용자 입력
-> Calendar Source Adapter
-> Privacy Filter
-> Availability Normalizer
-> Scheduling Engine
-> Agent Message Generator
-> 추천 후보 시간 + 공유 메시지
```

개인 일정 DB는 원본 캘린더 DB가 아니라 availability DB입니다. 일정 제목, 장소, 설명이 아니라 회의 조율에 필요한 busy/free block만 저장합니다.

## 5. 성과 평가 및 실행 계획

### MVP 범위

반드시 구현할 것:

- 새 회의 생성
- 참여자별 가능 시간 직접 입력
- 개인 일정의 busy/free block을 공통 availability DB로 변환
- `.ics` 또는 CSV 샘플 import 중 하나
- 30분 단위 공통 가능 시간 계산
- 후보 시간 3개 추천
- 추천 이유와 복사용 메시지 생성

이번에는 하지 않을 것:

- 외부 캘린더 OAuth 실시간 연동
- 로그인/회원가입
- 자동 메시지 발송
- 자동 일정 등록
- 반복 일정 최적화
- 원본 캘린더 이벤트 제목, 장소, 설명 저장

### 성공 기준

소마 팀원 3명과 멘토 1명의 1시간 회의 시간을 안정적으로 잡을 수 있으면 성공입니다.
