# 2026-05-01 2차 오프라인 회의록 보고용 초안

금일 5월 1일(금) 오전 10시에 팀원 5명이 오프라인으로 모여 2차 회의를 진행했습니다.

이번 회의의 목적은 5월 2일(토) 프로젝트 기획서 제출 전까지 기획서의 주요 항목을 구체화하고, 각자 작성한 내용을 통합하여 코치님께 피드백을 요청할 수 있는 단계까지 끌어올리는 것이었습니다.

먼저 팀명과 프로젝트명을 정리했습니다.

- 팀명: 소마밋
- 프로젝트명: 소마밋(SomaMeet)

1차 회의에서 정했던 “개인정보 보호가 되는 공유캘린더” 방향은 유지하되, 팀명과 프로젝트명을 모두 `소마밋(SomaMeet)`으로 통일하기로 했습니다.

이번 회의에서는 각자 준비한 기획 방향을 공유하는 방식도 정리했습니다.

각 팀원은 본인이 생각한 기획 방향을 먼저 작성한 뒤, main 브랜치에 PR을 올리고 병합하는 방식으로 사전 공유를 진행했습니다. 회의에서는 처음 발표한 사람의 내용과 각자 작성한 내용이 어떤 점에서 다른지, 추가로 보완할 지점이 무엇인지를 중심으로 발표했습니다.

이 방식 덕분에 단순히 같은 내용을 반복해서 발표하기보다, 팀원별 관점 차이와 보완점을 빠르게 확인할 수 있었고 회의도 깔끔하게 진행되었습니다.

Google Docs 기획서 작성 항목은 5개로 나누어 담당하기로 했습니다.
https://docs.google.com/document/d/1b8qOjApM89yzexKFhtl8Ey9evbAmEeR4l2QVjYCRcX4/edit?usp=sharing

1. 문제 정의 및 프로젝트 개요: 박세종
2. 사용자 및 Agent 설계: 안수빈
3. 핵심 기능 및 사용자 흐름: 지상근
4. 기술 구현 설계: 이중곤
5. 성과 평가 및 실행 계획: 홍지연

각자 담당한 항목은 Google Docs에서 구체화하고, 작성 중 논의가 필요한 포인트는 회의를 통해 합의했습니다. 
박세종은 각 팀원이 작성한 내용을 취합하여 전체 기획서 흐름을 통합하기로 했습니다.

기획서 제출까지 구체적인 일정은 다음과 같습니다.

- 5월 2일(토) 낮 12:00까지 각자 담당한 부분 작성 완료
- 5월 2일(토) 오후 2:00까지 박세종이 통합본 정리
- 통합본 정리 후 코치님께 피드백 요청
- 코치님 피드백 수신 후 최종 수정 - 박세종
- 5월 2일(토) 23:59 전 프로젝트 기획서 제출 완료 - 박세종

기술 스택은 아직 완전히 확정된 것은 아니지만, 현재는 아래 방향으로 임시 합의했습니다.

- Frontend: React
- Backend: Python, FastAPI
- Deployment: AWS Lightsail 4GB

기술 스택은 5월 2일 기획서 제출 전까지 완전히 고정하기보다, 데모 구현 가능성과 팀원별 개발 분담 가능성을 기준으로 계속 검토하기로 했습니다. 개발 역할 분담은 5월 3일(일) 온라인 회의에서 코치님 피드백을 받은 뒤 구체적으로 진행할 예정입니다.

현재는 개발 역할을 세부적으로 확정하기보다, MVP를 만들기 위해 필요한 큰 단위를 다음과 같이 보고 있습니다. 개발 일정은 5월 8일(금)까지 MVP를 완성하고, 이후 데모 코드 제출 전까지 안정화하는 방향으로 잡았습니다. MVP 완성 후에는 실제로 배포하여 사용자 테스트까지 진행해볼 예정입니다.

최종 발표와 관련해서는 5월 15일(금) 최종 발표에서 홍지연 님이 발표를 맡고, 박세종이 발표 준비와 진행을 서포트하기로 했습니다.


향후 주요 일정은 다음과 같습니다.

- 5월 3일(일) 19:00: 코치님과 온라인 회의 및 개발 역할 분담 논의
- 5월 8일(금): MVP 완성 목표
- 5월 10일(일) 23:59: 데모 코드 제출
- 5월 15일(금): 최종 발표 - 홍지연
