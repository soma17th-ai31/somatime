# SomaMeet 기획서 통합 레포

SW마에스트로 AI 기술교육 31조 소마밋 팀의 프로젝트 기획서를 모으고 통합하기 위한 문서 레포입니다.

현재 단계에서는 구현 코드를 두지 않습니다. 5명의 팀원이 각자 작성한 기획서를 `기획서/` 아래에 올리고, 2026-05-01(금) 회의에서 하나의 방향으로 통합합니다.

현재 최종 제출 기준 문서는 `기획서/구글독스_1차피드백_반영_통합본.md`입니다. AI 또는 협업자가 기획서 본문을 판단할 때는 이 파일을 우선해서 확인합니다.

## 현재 목표

- 2026-05-01(금): 팀원 5명의 개인 기획서 초안 공유
- 2026-05-02(토) 23:59: 프로젝트 기획서 제출
- 2026-05-10(일) 23:59: 데모 코드 제출
- 2026-05-15(금) 13:30-15:30: 최종 발표/데모데이

## 파일 구조

```text
.
├── README.md
├── 프로젝트_맥락.md
├── 전체일정.md
├── 작성가이드.md
└── 기획서/
    ├── raw/
    │   └── README.md
    ├── 공통_기획서_양식.md
    ├── 구글독스_복사용_기획서_양식.md
    └── 구글독스_1차피드백_반영_통합본.md
```

## 작성 방식

1. `기획서/raw/`의 개인 기획서와 raw 통합본은 원본 보관용으로 둡니다.
2. 1차 피드백을 반영한 최종 제출본은 `기획서/구글독스_1차피드백_반영_통합본.md`입니다.
3. Google Docs에 붙여넣을 때는 위 최종 제출본을 기준으로 사용합니다.

`기획서/raw/`에 들어간 원본 자료는 사람이 직접 관리하며, AI는 해당 폴더의 파일을 수정하지 않습니다.

원본 양식 설명은 `기획서/공통_기획서_양식.md`에 보관되어 있으며, 각 팀원별 템플릿에도 같은 설명을 포함했습니다.

## 프로젝트 맥락

회의 전에는 `프로젝트_맥락.md`에서 결정사항, MVP 범위, 열린 질문을 먼저 확인합니다. 제출 문서 본문은 `기획서/구글독스_1차피드백_반영_통합본.md`를 기준으로 합니다.

## 현재 가설

SomaMeet(소마밋)은 개인 일정 세부 내용을 공개하지 않고도 여러 사람의 공통 가능 시간을 빠르게 찾는 개인정보 보호 일정 조율 서비스입니다.

핵심 원칙:

- 원본 캘린더를 복제하지 않습니다.
- 일정 제목, 장소, 설명을 저장하지 않습니다.
- 회의 조율에 필요한 busy/free availability block만 사용합니다.
- 많은 기능보다 하나의 안정적인 일정 조율 흐름에 집중합니다.
