# SomaMeet 기획서 통합 레포

SW마에스트로 AI 기술교육 31조 소마밋 팀의 프로젝트 기획서를 모으고 통합하기 위한 문서 레포입니다.

현재 레포는 기획 문서와 회의 결과를 정리하는 용도로 사용합니다. 5명의 팀원이 각자 작성한 기획서는 `기획서/` 아래에 보관하고, 통합된 기준 문서는 `기획서/구글독스_1차피드백_반영_통합본.md`로 관리합니다.

현재 최종 제출 기준 문서는 `기획서/구글독스_1차피드백_반영_통합본.md`입니다. AI 또는 협업자가 기획서 본문을 판단할 때는 이 파일을 우선해서 확인합니다.

## 현재 목표

- 2026-05-05(화) 19:00: 각자 AI를 활용해 개발한 결과물을 코치님께 3~5분 발표
- 2026-05-06(수): 결과물 한계점과 코치님 피드백을 반영해 최종 역할 분담
- 2026-05-10(일) 23:59: 데모 코드 제출
- 2026-05-15(금) 13:30-15:30: 최종 발표/데모데이

개인 구현 결과물은 GitHub 조직의 개별 repository에 만들고 main 브랜치에 푸시합니다. 박세종 개인 구현 레포는 https://github.com/soma17th-ai31/SomaMeet_Sejong 입니다.

## 파일 구조

```text
.
├── README.md
├── 프로젝트_맥락.md
├── 전체일정.md
├── 작성가이드.md
├── 회의록/
└── 기획서/
    ├── raw/
    │   └── README.md
    ├── 공통_기획서_양식.md
    ├── 구글독스_복사용_기획서_양식.md
    └── 구글독스_1차피드백_반영_통합본.md
```

## 작성 방식

1. `기획서/raw/`의 개인 기획서와 raw 통합본은 원본 보관용으로 둡니다.
2. 1차 피드백을 반영한 최종 제출본은 `기획서/구글독스_1차피드백_반영_통합본.md`입니다.
3. Google Docs에 붙여넣을 때는 위 최종 제출본을 기준으로 사용합니다.

`기획서/raw/`에 들어간 원본 자료는 사람이 직접 관리하며, AI는 해당 폴더의 파일을 수정하지 않습니다.

원본 양식 설명은 `기획서/공통_기획서_양식.md`에 보관되어 있으며, 각 팀원별 템플릿에도 같은 설명을 포함했습니다.

## 회의록 관리

- 오프라인 회의 전에는 `회의록/YYYY-MM-DD_N차_오프라인_회의록_초안.md` 형식으로 회의록 초안을 제안합니다.
- 회의 후 보고용 내용이 확정되면 `회의록/YYYY-MM-DD_N차_오프라인_회의록_보고용.md`를 고정본으로 둡니다.
- 보고용 고정본이 생기면 같은 회차의 초안 파일은 삭제합니다.
- AI 또는 협업자가 회의 내용을 참고할 때는 초안보다 보고용 고정본을 우선합니다.
- 2026-05-01 2차 오프라인 회의의 고정본은 `회의록/2026-05-01_2차_오프라인_회의록_보고용.md`입니다.

## 프로젝트 맥락

회의 전에는 `프로젝트_맥락.md`에서 결정사항, MVP 범위, 열린 질문을 먼저 확인합니다. 제출 문서 본문은 `기획서/구글독스_1차피드백_반영_통합본.md`를 기준으로 합니다.

최근 온라인 코칭 결과는 `회의록/2026-05-03_2차_온라인_코칭_회의록.md`에 정리했습니다.

## 현재 가설

SomaMeet(소마밋)은 개인 일정 세부 내용을 공개하지 않고도 여러 사람의 공통 가능 시간을 빠르게 찾는 개인정보 보호 일정 조율 서비스입니다.

핵심 원칙:

- 원본 캘린더를 복제하지 않습니다.
- 일정 제목, 장소, 설명을 저장하지 않습니다.
- 회의 조율에 필요한 busy/free availability block만 사용합니다.
- 많은 기능보다 하나의 안정적인 일정 조율 흐름에 집중합니다.
